The Memory of a Potion

🧩 Game Programmer & Game and Level Designer

Media Gallery

Project Overview

Created a story puzzle game with clues around the environment. The game features puzzles that utilise environmental interactions and mechanics to challenge players. This was a solo project where I was responsible for all aspects of puzzle design, including concept development, prototyping, playtesting, and iteration. The goal was to create a cohesive puzzle experience that encourages exploration and using your surroundings to solve puzzles and remember your story.

Duration

1 month

Team Size

Solo Project

Roles

Game Programmer

Level Designer

Game Designer

Engine

Unity

Key Responsibilities

  • Designed the environmental interaction mechanics and systems
  • Created puzzles with clear visual and mechanical feedback
  • Prototyped puzzle concepts and iterated based on player feedback
  • Programmed puzzle mechanics and systems in Unity
  • Implemented a hint system that provides contextual clues without giving away solutions

Design Highlights

Elegant Mechanics

Core mechanics are simple to learn but allow for complex puzzle designs and fun solutions.

Story Integration

Puzzles reinforce the game's narrative, ensuring that solving them feels meaningful.

Hint System

Contextual hints provide helpful clues without revealing puzzle solutions outright.

Visual Clarity

Environmental design clearly communicates which objects are interactive and how.

Playtesting Insights

  • Clear visual feedback is essential for players to understand puzzle mechanics
  • Hint systems should be integrated naturally into the game world
  • Players prefer puzzles in their intended environment rather than abstracted versions
  • Making a puzzle feel like it belongs in the world is key to player engagement
  • Programming potion making in a way that is easy to expand and maintain is key to future development
